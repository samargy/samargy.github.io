Technical Aspects

Below is the directory structure for my site. It consists of a public folder - containing all non .css or .html resources, e.g. images and fonts. After the public folder there is the 4 HTML files, each with their own respective content. Then you have websystems.css which contains the CSS for the entire site, this means all the styling across the site uses the same style sheet. This is great for both developmental purposes (everything is changed at once) and also design purposes (consistency).

Now that the file structure has been layed out. Its time to talk a bit about the page structure itself. Below is a table of all my main CSS classes and ids and styles

Name Type Purpose
h1 element Styles the title at top of webpage
body element Styles the background of the site
.body-container class Styles the grey/yellow box most of my sites content lies in
.main-heading class Styles the "Comments" piece of text you see on this page. Could be changed to an ID, although kept as class for possible reuse.
.sub-heading class Styles the sub headings you see around the site, e.g Technical Aspects, Aesthetics.
#navbar ul id + element First using an ID to specify the element. Then styling the element. This part of the navbar styled is the unordered list containing its li elements.
#navbar ul li a id + element First using an ID to specify the element. Then styling the element. This part of navbar is each list item e.g Home, Past, Future. This styling also has another version of it with :hover added on which controls what styling occurs when the users mouse hovers over the list item.
